# Portal Controller - Инструкция по использованию

## Описание
Скрипт `PortalController` управляет анимацией портала с двумя состояниями:
1. **Анимация активации** - проигрывается один раз при включении портала
2. **Анимация idle** - зацикленная анимация активного портала

## Настройка в Unity

### 1. Создание портала
1. Создайте пустой GameObject в сцене (GameObject → Create Empty)
2. Назовите его "Portal"
3. Добавьте компонент `PortalController` (Add Component → Portal Controller)

### 2. Настройка анимаций

#### Анимация активации (Activation Sprites)
- Перетащите спрайты для анимации включения портала в массив `Activation Sprites`
- Эта анимация проиграется один раз при активации портала
- Если массив пустой, портал сразу перейдет к idle анимации

#### Анимация idle (Idle Sprites)
- Перетащите спрайты для зацикленной анимации в массив `Idle Sprites`
- Эта анимация будет проигрываться постоянно после завершения активации

#### Настройки анимации
- **Frame Rate** - скорость анимации в кадрах в секунду (по умолчанию 10)

### 3. Настройка активации

#### Параметры:
- **Activate After Task** - если включено, портал активируется после выполнения задания
- **Start Active** - если включено, портал активен с самого начала сцены

### 4. Интеграция с системой диалогов

Чтобы портал активировался после задания:

1. Откройте объект с компонентом `NPCDialogue` или `NPCDialogueChapter2`
2. Найдите нужную реплику в массиве `Dialogue Lines`
3. Включите галочку `Start Task After Line` на реплике с заданием
4. Включите галочку `Activate Portal After Task`
5. Перетащите ваш портал в поле `Portal To Activate`

## Использование из кода

### Активация портала
```csharp
PortalController portal = GetComponent<PortalController>();
portal.ActivatePortal();
```

### Деактивация портала
```csharp
portal.DeactivatePortal();
```

### Проверка состояния
```csharp
// Проверить, активен ли портал (и завершена ли анимация активации)
if (portal.IsActive())
{
    Debug.Log("Портал активен!");
}

// Проверить, проигрывается ли анимация активации
if (portal.IsActivating())
{
    Debug.Log("Портал активируется...");
}
```

## Пример настройки

### Сценарий: Портал открывается после выполнения задания

1. **Создайте портал:**
   - GameObject → Create Empty → "Portal"
   - Add Component → Portal Controller
   - Add Component → Sprite Renderer (добавится автоматически)

2. **Добавьте спрайты:**
   - Activation Sprites: portal_open_01, portal_open_02, portal_open_03
   - Idle Sprites: portal_idle_01, portal_idle_02, portal_idle_03, portal_idle_04

3. **Настройте параметры:**
   - Frame Rate: 12
   - Activate After Task: ✓
   - Start Active: ✗

4. **Настройте диалог:**
   - Откройте NPC с заданием
   - На реплике с заданием:
     - Start Task After Line: ✓
     - Activate Portal After Task: ✓
     - Portal To Activate: [перетащите Portal]

5. **Готово!** Портал активируется после выполнения задания

## Визуализация в редакторе

В режиме Scene портал отображается как:
- **Серая сфера** - портал неактивен
- **Голубая сфера** - портал активен

## Советы

1. **Оптимизация:** Используйте спрайты одинакового размера для плавной анимации
2. **Скорость:** Подбирайте Frame Rate в зависимости от количества кадров (обычно 8-15 fps)
3. **Тестирование:** Включите Start Active для быстрого тестирования анимаций
4. **Порядок слоев:** Настройте Order in Layer в Sprite Renderer для правильного отображения

## Troubleshooting

**Проблема:** Портал не появляется
- Проверьте, что Sprite Renderer включен
- Убедитесь, что добавлены спрайты в массивы
- Проверьте, что Start Active включен (для теста)

**Проблема:** Анимация не проигрывается
- Проверьте Frame Rate (должен быть > 0)
- Убедитесь, что в массивах есть спрайты
- Проверьте, что портал активирован (вызван ActivatePortal)

**Проблема:** Портал не активируется после задания
- Проверьте галочку Activate Portal After Task
- Убедитесь, что Portal To Activate заполнен
- Проверьте, что задание действительно выполняется
